# Big-O Analysis (Day 1)

## What is Big-O?
Big-O describes how an algorithm's running time or memory usage grows as the input size increases.

## Time Complexity
Measures how the number of operations grows with input size.

## Space Complexity
Measures how much extra memory an algorithm uses.

## Common Complexities
- O(1)
- O(log n)
- O(n)
- O(n log n)
- O(n²)

## Rules
- Ignore constants.
- Keep the highest-order term.
- Analyze the worst case unless stated otherwise.

## Examples
- Single loop → O(n)
- Nested loops → O(n²)
- Loop doubling each time → O(log n)
